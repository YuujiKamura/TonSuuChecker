# PR作成コマンド

現在のブランチの変更内容からPull Requestを作成してください。

## 手順

1. `git log` で現在のブランチのコミット履歴を確認
2. `git diff` で変更ファイルを確認
3. 変更内容を分析してPRのタイトルと本文を作成
4. `gh pr create` でPRを作成（ghコマンドが使えない場合はPR作成用URLを提示）

## PR本文のフォーマット

```markdown
## Summary
- 変更内容を箇条書きで記載

## Test plan
- [ ] テスト項目をチェックリストで記載
```

## 注意事項

- mainブランチへのPRを想定
- コミットメッセージから変更内容を読み取る
- 日本語で記述
